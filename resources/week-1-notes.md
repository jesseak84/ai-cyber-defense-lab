## What I Learned

- A local Git repository lives on my computer.
- GitHub is the remote online version.
- I need to be inside the correct project folder before running Git commands.
- `git add` stages files.
- `git commit` saves a version locally.
- `git push` uploads commits to GitHub.
- Documentation is part of cybersecurity work, not an afterthought.

## Problems Encountered

- I accidentally created the repo in the wrong folder first.
- I learned that my Desktop path is under OneDrive.
- I tried running Git commands outside the repo and got `fatal: not a git repository`.
- I accidentally combined commands when typing `git pushcd`.

## Fixes

- Moved the repo to the correct OneDrive Desktop Cyber Fu folder.
- Used `cd` to enter the repo before running Git commands.
- Re-ran the Git commands from the correct location.

## Defensive / Professional Takeaway

Cybersecurity work requires precision. Being in the wrong folder, running the wrong command, or assuming the wrong context can create bad results. This applies to Git, investigations, logs, evidence handling, and incident response.
